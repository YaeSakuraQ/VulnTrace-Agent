# 演示任务模板

系统内置了 2 个课程展示模板，前端首页可直接点击套用：

- 每个模板都带有推荐 IP/范围快捷填充。
- 每个模板都带有推荐端口预设，可直接覆盖表单。
- 选中模板后，页面会显示“课堂演示顺序”，便于按步骤展示。

## 1. DVWA

- 适合展示 Web 面的低风险探测、目录检查、审批式验证，以及单条受控 PoC 的证据链。
- 默认范围是 `127.0.0.1`，适合 Docker 端口映射。
- 如果 DVWA 在独立虚拟机里，建议改成 `192.168.56.x` 这类 Host Only IP。
- 默认目标已经调整为“审批后执行单条受控漏洞 PoC，并在报告里写清请求、参数和成功证据”。

## 2. Vulhub

- 适合 Docker 单漏洞场景，通常使用 `127.0.0.1` 和映射端口。
- 模板会提示你先核对 compose 暴露端口，再运行任务。
- 很适合做“审批前后对比”和“证据驱动报告”的演示。

## 使用建议

- 套用模板后，先改成你真实靶场的 IP/端口。
- 如果只是课堂演示完整闭环，可直接使用“按模板创建并运行”按钮。
- 如果 IP 或端口需要微调，直接点击模板下方的快捷入口即可，不必手工重输。
- 演示前先手动确认目标可访问，避免把“靶场没起来”误当成智能体问题。
- 汇报时优先展示模板说明、阶段推进、审批卡片，以及最终报告中的 PoC 请求/参数/证据段。
